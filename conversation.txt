[7] 부팅 후 서비스 자동시작 설정 (.env 파일 + 서비스 파일 개선)
─────────────────────────────────────────────────────────────

사용자:
  리부팅하고 나니까 서비스가 자동 실행되지 않고 서비스가 중지되어 있는데,
  리부팅되더라도 서비스가 자동 실행하도록 환경파일을 만들어 주세요.

Claude:
  원인: systemctl enable 미실행 또는 서비스 파일 미등록 추정.
  추가로 서비스 파일의 재시작 정책 및 의존성 조건을 강화.

  신규 생성 — .env:
    OLLAMA_HOST=http://localhost:11434
    OLLAMA_MODEL=qwen3-vl:8b
    REQUEST_TIMEOUT=180
    MAX_IMAGE_BYTES=10485760
    APP_PORT=8001

  qwen3-vl.service 변경 사항:
    - Environment= 인라인 → EnvironmentFile=.env 파일 분리
    - After=network.target → After=network-online.target
      (네트워크 완전 연결 후 기동 보장)
    - Wants=network-online.target 추가
    - Restart=on-failure → Restart=always
      (어떤 이유로든 종료 시 재시작)
    - TimeoutStartSec=30 → 60 (부팅 직후 Ollama 로드 시간 여유)

  배포 단계:
    1) .env 파일 운영 경로 복사
         cp ~/qwen3-vl/.env /var/www/atpia.com/public_html/qwen3-vl/.env
    2) 업데이트된 서비스 파일 복사
         sudo cp ~/qwen3-vl/qwen3-vl.service /etc/systemd/system/
    3) systemd 리로드 + 부팅 자동시작 등록
         sudo systemctl daemon-reload
         sudo systemctl enable --now qwen3-vl

  확인:
    curl http://127.0.0.1:8001/api/health
    → {"ollama":"ok","target_model":"qwen3-vl:8b","model_ready":true,
       "available_models":["qwen3-vl:8b","qwen3:14b","qwen3:8b"]}

  리부팅 후 https://atpia.com/qwen3-vl/index.html 접속 정상 확인.
